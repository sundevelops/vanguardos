## 2. Concrete improvements (4)

1. **~94 MB → ~230 KB of page images.** The catalog rendered three ~20–27 MB PNG thumbnails plus a 21 MB banner. On a 10 Mbps connection that's a minute-plus of loading below the fold and a guaranteed Core Web Vitals failure (LCP/total weight), which suppresses both Google ranking and conversion. All four page images now load as right-sized WebP: bundle banner 110 KB, thumbnails 31–54 KB. Visual quality at display size is indistinguishable (q82, Lanczos downscale from the 4K masters).
2. **Social-share/SEO meta repaired.** Description, og:title and twitter:title still sold "20 minutes" — off-brand and contradicting the on-page 60-minute promise (a credibility leak the moment someone shares the link). og:image pointed at a 6.8 MB 2048² PNG, which Twitter/LinkedIn/WhatsApp scrapers reject or crop badly (most cap ~5 MB). Now: correct 60-minute copy, the $5K–$10K/month anchor in the og:description (Hormozi price-anchor working even in a link preview), and a purpose-built 1200×630 110 KB card with width/height/alt tags so previews render instantly and correctly everywhere.
3. **Upsell modal a11y + real Esc bug fix.** The overlay was `role="button"` wrapping a `role="dialog"` (invalid semantics, triggered svelte-check warnings), and the card's `on:keydown|stopPropagation` swallowed keydown events before they reached the window listener — meaning **Esc didn't close the modal while focus was inside it** (e.g. after tabbing to the checkbox or buttons). Removed the keydown trap, cleaned the roles, kept click-outside-to-close, added explicit svelte-ignore markers for the intentional overlay click handler.
4. **Layout-shift (CLS) guard.** Bundle banner and catalog images now declare explicit `width`/`height`, so the browser reserves space before images arrive — no content jump in the catalog/bundle-reveal sections while scrolling. CLS is a direct Core Web Vitals ranking factor.
