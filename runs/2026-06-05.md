## 2. Concrete improvements (4)

1. **Full static prerender (SEO).** Until now the deployed page was a client-rendered SvelteKit app — a crawler or social scraper that doesn't execute JS got `<div style="display: contents"></div>` and nothing else. The build now emits `prerendered/pages/index.html` (78 KB of complete HTML: hero copy, all six chat logs, FAQ, schema.org JSON-LD, every CTA). Google indexes the real page, link previews and non-JS agents see real content, and first paint no longer waits on the JS bundle. Verified by inspecting the prerendered file.
2. **Build/deploy slimmed ~200 MB → ~700 KB.** Every `npm run build` was copying ~199 MB of unreferenced 4K PNG masters (products, brand profiles, the old gumroad thumbnails) into `output/client/`, ballooning deploys and leaving 26 MB PNGs publicly hot-linkable next to their 35 KB WebP twins. Verified via grep that zero references exist in `src/`, then moved the masters to `assets-src/` with a README. `output/client` is now 672 KB total. (The 2026-06-04 log explicitly queued this; the README preserves the "kept just in case" intent.)
3. **Gold scroll-progress hairline (godly.website pass).** Completed the queued godly scrape — the through-line on its top-rated product sites (Reflect, Amie, AuthKit, Atlas) is restrained scroll affordance, not more motion. Added a 2px fixed gold-gradient progress bar across the top of the viewport (brand gradient `#B8941F → #D4AF37 → #E7C66A`, soft glow, `transform: scaleX` so it never triggers layout). On a long single-page funnel it gives buyers a constant sense of "how much is left," which fights mid-page abandonment. Zero changes to existing animations.
4. **Upsell modal keyboard flow fixed.** Opening the modal previously left focus on the CTA *behind* the backdrop — a keyboard user's next Tab moved through the hidden page, and Esc-closing stranded focus. Now: open → focus lands on "Continue to Checkout" (one keystroke from converting); close/Esc → focus returns to the exact CTA that opened it. Also unified the Esc path through `upsellClose()` so both dismissal routes behave identically.

Plus a micro credibility fix: body copy said "the other 209 emails," the chat log next to it proved 208 (211 unread − 3 urgent). The skeptical buyer who does that arithmetic is exactly the one deciding whether to trust you. Now consistent at 208.
